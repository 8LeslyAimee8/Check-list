Name: Checklist example

Description: This file contains 6 sections of test cases covering different screens of a mobile app. It includes:

* Collection Point: 8 test cases.

* Dish Selection: 18 test cases.

* Order Confirmation: 11 test cases.

* Order Tracking: 12 test cases.

* Order Sent & Error Notification screens.
It also features a description for each test, a status column, and links to Jira/Error reports.

Method: After analyzing the requirements for each screen, I drafted and organized the test cases into an initial checklist, ensuring every flow was listed and verified.

Tools: Google Sheets for team accessibility and a clean, easy-to-track layout. Jira for logging and managing identified bugs.

Results: After executing 60+ test cases, I identified 2 bugs and successfully documented them in Jira.
